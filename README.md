# Eve Craft

Eve Craft - настольное приложение для работы с данными и рабочими сценариями EVE Online. Проект развивается как доменно-ориентированный модульный монолит: оболочка приложения, бизнес-модули industry и market, а также общие технические сервисы находятся в одной кодовой базе, но с явными границами ответственности.

Сейчас в репозитории уже есть запуск приложения, основная Qt-оболочка, локальная runtime-инфраструктура и платформенные сервисы для настроек, логирования, баз данных и синхронизации SDE.

## Описание проекта

Приложение построено вокруг структуры `src/eve_craft` и разделено на четыре основные зоны:

- `app` - bootstrap, контейнер зависимостей, навигация и главное окно
- `modules` - бизнес-модули `industry` и `market`
- `platform` - общие технические сервисы: `db`, `sde`, `auth`, `settings`, `logging`
- `shared` - небольшие переиспользуемые примитивы

Локальные рабочие данные создаются в папке `runtime/`: базы данных SQLite, временные файлы, загрузки, настройки и логи.

## Стек технологий

- Python 3.13
- PySide6 для desktop-интерфейса
- SQLite для локального хранения данных
- SQLAlchemy 2 для работы с базой данных
- `unittest` для автотестов

## Структура проекта

```text
src/eve_craft/
  app/         bootstrap приложения, контейнер, навигация, main window
  modules/     бизнес-модули industry и market
  platform/    общие технические сервисы
  shared/      небольшие общие примитивы
  resources/   ресурсы пакета
```

## Установка зависимостей

После активации виртуального окружения установите зависимости проекта:

```powershell
pip install -r requirements.txt
```

## Запуск приложения

Запуск из корня репозитория:

```powershell
python main.py
```

При первом старте приложение создаст папку `runtime/`, локальные базы SQLite и ресурсный каталог. Если SDE-ресурсы еще не подготовлены, стартовый сценарий сможет инициализировать или обновить и базу SDE, и локальный набор `IEC Types`-изображений.

## `.env` для ESI SSO

Для dev-режима приложение автоматически загружает переменные окружения из файла `.env` в корне проекта до построения конфигурации приложения.

Поддерживаемые ключи для ESI SSO:

- `EVE_CRAFT_ESI_CLIENT_ID`
- `EVE_CRAFT_ESI_CLIENT_SECRET`
- `EVE_CRAFT_ESI_CALLBACK_URL`

Шаблон лежит в `.env.example`. Для вашего текущего callback используйте:

```dotenv
EVE_CRAFT_ESI_CLIENT_ID=your_client_id
EVE_CRAFT_ESI_CLIENT_SECRET=your_client_secret
EVE_CRAFT_ESI_CALLBACK_URL=http://127.0.0.1:8080/callback
```

Если те же переменные уже заданы в системном окружении, они имеют приоритет над `.env`.

## Запуск тестов

Полный прогон тестов:

```powershell
py -3.13 -m unittest discover -s tests -t . -v
```

Запуск отдельного тестового модуля:

```powershell
py -3.13 -m unittest tests.platform.sde.test_synchronizer -v
```
