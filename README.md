# IPH2

Проект хранится в папке, которая синхронизируется через Яндекс.Диск. Поэтому виртуальное окружение Python не нужно хранить внутри проекта и не нужно переносить между устройствами.

Правильный подход:

- проект и код синхронизируются через Яндекс.Диск или Git;
- на каждом компьютере создается свое локальное окружение;
- зависимости ставятся из `requirements.txt`.

## Рекомендуемая схема

Создавайте виртуальное окружение вне папки проекта.

Примеры:

- Windows: `C:\Users\<имя>\.venvs\IPH2`
- macOS: `~/.venvs/IPH2`

Это защищает проект от проблем с абсолютными путями, разными ОС и сломанными launcher-файлами.

## Windows PowerShell

Проверьте установленный Python:

```powershell
py -0p
```

Создайте окружение:

```powershell
py -3.13 -m venv $HOME\.venvs\IPH2
```

Активируйте его:

```powershell
$HOME\.venvs\IPH2\Scripts\Activate.ps1
```

Установите зависимости проекта:

```powershell
python -m pip install --upgrade pip
pip install -r requirements.txt
```

После активации окружения можно запускать инструменты проекта, например:

```powershell
pyside6-designer
```

## macOS

Проверьте, что нужный Python установлен:

```bash
python3.13 --version
```

Создайте окружение:

```bash
python3.13 -m venv ~/.venvs/IPH2
```

Активируйте его:

```bash
source ~/.venvs/IPH2/bin/activate
```

Установите зависимости проекта:

```bash
python -m pip install --upgrade pip
pip install -r requirements.txt
```

## Повседневная работа

При каждом новом терминале:

1. Перейдите в папку проекта.
2. Активируйте локальное окружение.
3. Работайте с проектом.

## Если окружение сломалось

Если проект был перенесен в другую папку, открыт на другом ПК или после обновления Python появились ошибки вида `Unable to create process`, просто удалите локальное окружение и создайте его заново на текущем устройстве.

Важно: удалять нужно только локальное окружение, а не файлы проекта.
